# User Workflows

## 1. Create or Open a Project

Users work inside lightweight Projects from the Side Panel. The extension supports a maximum of 5 Projects to keep the MVP focused and simple.

Each Project contains Prompt Templates, Prompt Runs, Outputs, Source Images, and an Artifact Library.

## 2. Build a Prompt Template

Users create node-based Prompt Templates on a canvas. Nodes represent reusable prompt sections such as product context, visual style, brand rules, composition, aspect ratio, negative prompts, and output requirements.

Users connect prompt nodes to define the final prompt assembly order.

## 3. Generate a Final Prompt

The prompt graph engine combines connected nodes into a final prompt preview. Users can edit the final prompt before sending it to an external tool.

## 4. Send to an AI Tool

The MVP supports semi-automatic handoff to ChatGPT, Gemini, and Google Flow. Users remain responsible for submitting, confirming, solving captchas, signing in, and following tool terms.

## 5. Attach and Review Outputs

Generated images are attached back to a Prompt Run as temporary Outputs. Users can review, annotate, edit metadata, or reject Outputs.

## 6. Approve Artifacts

An Output becomes an Artifact only after explicit user approval. Artifacts are reusable final assets stored in the Project Artifact Library.
